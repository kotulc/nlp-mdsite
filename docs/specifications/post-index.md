# Post Index

`PostIndex` fetches `public/posts-index.json` at runtime and renders a reverse-chronological
listing of all dated pages. Embed it in any MDX page with `<PostIndex />`. The data is
generated by `scripts/ingest.js` from any page with a `date` frontmatter field.

## Requirements

1. REQ-1: All posts from `posts-index.json` are rendered as list entries
2. REQ-2: Each entry shows the post title as a link to its URL
3. REQ-3: Each entry shows the formatted date (`Month D, YYYY`) when present
4. REQ-4: Each entry shows reading time in minutes when present
5. REQ-5: Each entry shows category chips (`chip-category`) and tag chips (`chip-tag`) when present
6. REQ-6: Posts are rendered in the order returned by the index (ingest sorts newest-first)
7. REQ-7: A loading state is shown while the fetch is in progress
8. REQ-8: A "No posts found" message is shown if the index returns an empty array

## Test Cases

`tests/components/PostIndex.test.jsx`

- `test_post_index_renders_all_posts` — all post titles rendered after fetch (REQ-1)
- `test_post_index_shows_reading_time` — reading time shown for each post (REQ-4)
- `test_post_index_shows_category_chips` — category chips rendered with correct class (REQ-5)
- `test_post_index_preserves_order` — posts appear newest-first as returned by index (REQ-6)
