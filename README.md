# nlp-mdsite
A Next.js powered static site template for intelligent, agent-friendly publishing from markdown.


## Purpose
`nlp-mdsite` provides a minimal, mobile-first site template designed to work at the end of a
markdown publishing pipeline. The goal is a push-button workflow: point the pipeline at a
directory of markdown files, run a single command, and get a fully built, deployed website.

The template is designed to be configured as easily by an AI agent as by a human — a small
config change is all that's required to launch a new site.


## How It Works
1. Write content as markdown files
2. Run the `mdpub` pipeline to process, enrich, and export content
3. `nlp-mdsite` builds and packages the static site via Next.js
4. Deploy to GitHub Pages (or any static host) via a single command


## Features
- MDX-based static site generation (Next.js + Nextra)
- Minimalist mobile-first layout with project and folder-level routing
- Page metadata (tags, metrics) surfaced directly in the UI
- Semantic search built in as a default component
- Semantic theming — color palette and visual identity derived from content signals
- Simple config-file driven setup, compatible with agent-based configuration
- GitHub Actions deployment workflow


## Intelligence Layer
The "NLP" in `nlp-mdsite` refers to a set of built-in intelligent features:

- **Semantic search** — readers can search across all content by meaning, not just keywords
- **Semantic theming** — the site's visual identity (color palette, logo hints) is automatically
  derived from content signals rather than manually specified
- **Rich metadata display** — tags, reading metrics, and other signals generated by the upstream
  pipeline are displayed as first-class UI elements on each page

These features are intended as defaults, not plugins. The upstream pipeline (`mdpub`) is
responsible for generating the metadata and semantic signals; this template consumes and
renders them.


## Integrations
`nlp-mdsite` is the rendering and build step at the end of the [`mdpub`](https://github.com/kotulc/nlp-mdpub)
publishing pipeline. Configuration, API interfaces, and MCP tooling live in `mdpub`;
this template is driven entirely by its config file and content directory.

- `mdpub init` installs dependencies and copies the template to the destination
- `mdpub build` / `preview` / `publish` drive the site lifecycle


## Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) 18 or later
- npm (bundled with Node.js)

### Setup

**1. Install dependencies**
```bash
npm install
```

**2. Add your content**

Place your markdown files in a source directory in any folder structure. The directory tree
is mirrored directly into the site — subdirectories become URL path segments.

```
<source>/
├── home.md          # home.md or index.md at any level → section landing page
├── about-us.md
├── images/          # images/ next to any .md files → copied to public/images/
└── posts/           # any subdirectory structure is valid
    └── <year>/
        ├── post-slug.md
        └── images/
```

Frontmatter fields used by the template: `title`, `date`, `categories`, `tags`.

**3. Configure the site**

Edit [site.config.js](site.config.js) to set the site title and base URL:
```js
module.exports = {
  title: 'My Site',
  base_url: 'https://username.github.io',
  base_path: '/repo-name',   // empty string for root domains
}
```

**4. Run setup**
```bash
npm run setup                             # ingest from examples/frww (default)
npm run setup -- --source path/to/content # ingest from a custom directory
npm run setup -- --build                  # also run a production build
```

This validates your environment, copies markdown into `pages/`, renames `.md` to `.mdx`,
copies images to `public/images/`, rewrites image paths, fixes any corrupt JPEG EXIF data,
and auto-generates `_meta.json` ordering files and `posts-index.json` from frontmatter dates.

**5. Preview locally**
```bash
npm run dev        # development server with hot reload
npm run build      # production build → out/
npm run preview    # serve the built out/ directory
```
